# Web presentación Act_2_EEDD

Proyecto sencillo (web en HTML y CSS) para la Actividad 2 de la asignatura Entornos de Desarrollo.  
El objetivo es practicar un flujo real de trabajo con Git: staging, commits, historial, checkout, ramas, merge, resolución de conflictos, etiquetas y (opcionalmente) GitHub.

## Archivos del proyecto
- `index.html`: página principal
- `style.css`: estilos CSS
- `README.md`: descripción del proyecto y pasos de la práctica

## Pasos que pide la práctica (resumen)
1. Inicialización del repositorio (creación de carpeta, `git init` y configuración de usuario).
2. Área de trabajo y zona de preparación (staging area): crear/modificar archivos y usar `git add`.
3. Creación de commits: registrar cambios con `git commit`.
4. Exploración del historial: consultar commits con `git log` y `git log --oneline`.
5. Navegación entre versiones: moverse entre commits con `git checkout`.
6. Ramas y fusiones: crear ramas, trabajar en ellas y fusionar con `git merge`.
7. Resolución de conflictos (si aparece durante un merge).
8. Etiquetado de versiones: crear tags con `git tag`.
9. GitHub (opcional): conectar remoto y realizar `push` y `pull`.
